# Project Demo Video Planning

**Project:** Automated Employee Onboarding & Offboarding System

---

## Description

This document outlines the structure and key steps for the **project demonstration video**. The demo will showcase the end-to-end functionality of the automated employee onboarding and offboarding system built on ServiceNow, including request submission, approvals, task generation, SLA monitoring, and lifecycle closure.

---

# Demo Flow

## 1. Service Portal Walkthrough

### Navigation

Service Portal → **Service Catalog → HR Services → Onboard / Offboard Employee**

### Demonstration Points

* Show the Service Portal interface.
* Navigate to the **HR Services category**.
* Open the **Onboard / Offboard Employee** catalog item.
* Explain the purpose of the catalog form.

---

## 2. Request Submission

### Steps

1. Select **Request Type** (Onboarding or Offboarding).
2. Fill in the required fields:

   * Employee ID
   * Department
   * Manager
   * Joining Date / Exit Date
   * Asset Details
   * Access Requirements
3. Submit the request.

### Demonstration Points

* Show dynamic form behavior using **UI Policies**.
* Display how fields change based on request type.

---

## 3. Manager Approval Process

### Demonstration Points

* Show how the request is routed to the **manager for approval**.
* Display the approval request in the manager's interface.
* Approve the request to trigger the next workflow steps.

---

## 4. Departmental Task Creation

After approval, the system automatically generates tasks for:

* HR
* IT
* Facilities
* Security

### Demonstration Points

* Show tasks generated for each department.
* Display task assignment and progress tracking.

---

## 5. SLA Monitoring

### Demonstration Points

* Show SLA timers attached to departmental tasks.
* Explain how SLAs help track task completion deadlines.
* Demonstrate SLA status updates.

---

## 6. Notifications

### Demonstration Points

Show automated **email notifications** sent to:

* Managers for approval requests
* Departments for task assignments
* Employees for status updates
* Request closure confirmation

---

## 7. Request Lifecycle Completion

### Demonstration Points

* Show completion of all departmental tasks.
* Demonstrate automatic **status updates in the Employee Lifecycle table**.
* Show final request closure.

---

# End-to-End Lifecycle Summary

The demo video should clearly show the complete lifecycle:

1. Request Submission
2. Manager Approval
3. Task Creation
4. SLA Monitoring
5. Notifications
6. Task Completion
7. Request Closure

---

## Expected Outcome of Demo

The demonstration should clearly show:

* Automation of employee lifecycle processes.
* Coordination between HR, IT, Facilities, and Security teams.
* Workflow transparency and lifecycle tracking.
* Improved efficiency through ServiceNow automation.
