# Usability

**Project:** Automated Employee Onboarding & Offboarding System
**Phase:** Usability & User Experience

---

## Description

The system improves usability by dynamically displaying form fields based on the user's selection. When a user selects a specific request type (Onboarding or Offboarding), only the relevant fields appear on the form. This reduces confusion, prevents incorrect inputs, and improves the overall user experience.

---

## Dynamic Field Behavior

The form fields are displayed based on the **Request Type** selected by the user.

### Case 1: Onboarding Selected

If the user selects **Request Type = Onboarding**, the following fields will be displayed:

* Employee Name
* Employee ID
* Department
* Joining Date
* Manager Name

These fields collect the required details for onboarding a new employee.

---

### Case 2: Offboarding Selected

If the user selects **Request Type = Offboarding**, the following fields will be displayed:

* Employee Name
* Employee ID
* Department
* Exit Date
* Reason for Exit

These fields capture the necessary details for employee offboarding.

---

## Benefits

* Simplifies the request submission process.
* Displays only relevant fields to the user.
* Reduces chances of incorrect data entry.
* Improves the overall user experience in the Service Portal.

---

## Expected Outcome

* Users can easily submit onboarding or offboarding requests.
* Forms become simpler and easier to understand.
* Data collection becomes more accurate and efficient.
